# Minimal Starter Template — Design Doc

**Date:** 2026-03-10
**Status:** Approved
**Approach:** Faithful deathcookies clone, stripped to minimum

## Summary

Build a minimal full-stack starter template for yomauro: Go backend + Next.js frontend following deathcookies conventions. Single bootstrapped owner account, session-based login, and a dashboard page displaying "Yo Mauro" in large text.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Roles | Single `owner` role, bootstrapped from env vars | Minimal — no user management UI needed |
| Multi-tenancy | None — no `tenant_id` | Simplest schema; add later if needed |
| Password reset | None — no email infrastructure | Change password via DB directly |
| Navigation | Sidebar shell (Dashboard + Logout) | Ready to add nav items later |
| Cookie prefix | `ym_session` / `ym_user` | Project-specific naming |
| Migration tool | golang-migrate | Same as deathcookies |
| Makefile | Single root Makefile, all entrypoints | Convention: all operations invoked via `make` |

## Database Schema

### users

```sql
CREATE TABLE users (
    id            SERIAL PRIMARY KEY,
    email         TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    display_name  TEXT NOT NULL,
    role          TEXT NOT NULL DEFAULT 'owner' CHECK (role IN ('owner')),
    active        BOOLEAN NOT NULL DEFAULT true,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### sessions

```sql
CREATE TABLE sessions (
    id         TEXT PRIMARY KEY,
    user_id    INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at TIMESTAMPTZ NOT NULL
);
CREATE INDEX idx_sessions_user_id ON sessions (user_id);
CREATE INDEX idx_sessions_expires_at ON sessions (expires_at);
```

No `tenant_id`, no `password_resets` table.

## Go Backend

### Entrypoint (cmd/api/main.go)

1. Load config from env vars via `internal/config`
2. Connect to PostgreSQL (pgxpool)
3. Bootstrap owner if none exists (from ADMIN_EMAIL / ADMIN_PASSWORD)
4. Build chi router via `handler.NewRouter()`
5. Start HTTP server on LISTEN_ADDR, graceful shutdown on SIGINT/SIGTERM

### Auth Package (internal/auth/)

- `HashPassword(password) (string, error)` — bcrypt cost 12
- `CheckPassword(password, hash) error` — bcrypt compare
- `DummyHash() string` — timing-attack prevention (sync.Once)
- `GenerateSessionToken() (string, error)` — 32 bytes, hex-encoded (64 chars)
- `RoleLevel(role) int` — owner = 3 (single value for now)
- `BootstrapOwner(ctx, queries, email, password, logger) error` — creates first owner if CountActiveOwners() == 0
- `RateLimiter` — IP-based, configurable max attempts and window

### Config Package (internal/config/)

`Settings` struct:
- `DatabaseURL string`
- `ListenAddr string` (default `:8080`)
- `AuthSecret string`
- `CORSOrigins string` (default `*`)
- `AdminEmail string`
- `AdminPassword string`
- `Environment string` (default `development`)

Startup validation (non-dev): fail if AUTH_SECRET is dev default or DATABASE_URL is empty.

### Handler Package (internal/handler/)

Router middleware stack: RequestID → Recovery → Logging → CORS

Routes:
- `POST /api/v1/auth/login` — public, rate-limited
- `GET /api/v1/auth/me` — SessionAuthMiddleware
- `POST /api/v1/auth/logout` — SessionAuthMiddleware

`AuthUser` struct in request context. `GetAuthUser(ctx)` helper.

Session cookie: `ym_session`, HttpOnly, Secure, SameSite=Lax, MaxAge=7 days. Sliding window (extend on each authenticated request).

### Repository (internal/repository/)

sqlc queries in `queries/auth.sql`:
- `GetUserByEmail` — lookup by email
- `CountActiveOwners` — for bootstrap check
- `CreateUser` — insert new user
- `CreateSession` — insert session with expiry
- `GetSession` — join sessions + users, check expires_at > NOW()
- `ExtendSession` — update expires_at
- `DeleteSession` — single session (logout)
- `DeleteUserSessions` — all sessions for user

## Next.js Frontend

### Middleware (middleware.ts)

Public paths: `/login`, `/api/*`, `/_next/*`, `/healthz`

Flow:
1. Read `ym_session` cookie
2. If missing → redirect `/login`
3. Check `ym_user` cache cookie (5-min TTL)
4. If expired → validate against Go backend `GET /api/v1/auth/me` via INTERNAL_API_URL
5. On failure → redirect `/login`, clear cookies
6. On success → set `x-user-id`, `x-user-role` request headers

### Auth Layout (app/(auth)/layout.tsx)

Centered flex container, light gray background. "Yo Mauro" branding text, max-width 448px.

### Login Page (app/(auth)/login/page.tsx)

Renders `<LoginForm />`. Email + password fields, submit button. POST to `/api/v1/auth/login`. On success: `router.push("/")` + `router.refresh()`. Error display for invalid credentials.

### Dashboard Layout (app/(dashboard)/layout.tsx)

Sidebar (16rem, desktop) + main content area. Mobile bottom tab bar.

### Dashboard Page (app/(dashboard)/page.tsx)

Large centered "Yo Mauro" text.

### Sidebar (components/layout/Sidebar.tsx)

Desktop: fixed left sidebar, 16rem width. Dashboard link with active highlight. Logout button at bottom with border-t separator. Mobile: bottom tab bar with safe area inset. Role-based filtering ready (single item for now).

### API Proxy (app/api/v1/[...path]/route.ts)

Catch-all proxy to Go backend (INTERNAL_API_URL). Forwards cookies, returns Set-Cookie headers. Uses AUTH_SECRET for server-side bearer auth.

### API Client (lib/api.ts)

- `login(email, password)` → POST `/api/v1/auth/login`
- `logout()` → POST `/api/v1/auth/logout`
- `me()` → GET `/api/v1/auth/me`
- `ApiError` class for non-2xx

### Styling

Tailwind CSS, no component library. Colors: gray-900 primary text, gray-50/100 backgrounds. Custom spacing: `sidebar: 16rem`. Inter font.

## Configuration

### .env.example

```
# Server
LISTEN_ADDR=:8080
ENVIRONMENT=development

# Database
DATABASE_URL=postgres://localhost:5432/yomauro?sslmode=disable

# Auth
AUTH_SECRET=dev-secret-change-me

# CORS
CORS_ORIGINS=http://localhost:3000

# Bootstrap (first run only)
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=changeme
```

### Frontend env (web/.env.local)

```
NEXT_PUBLIC_API_URL=http://localhost:8080
INTERNAL_API_URL=http://localhost:8080
```

## Makefile Targets

All operations go through the root Makefile:

- `make dev` — run Go server (go run or air)
- `make build` — compile Go binary
- `make migrate` — run pending migrations
- `make migrate-create` — create new migration pair
- `make sqlc` — regenerate sqlc code
- `make web-dev` — run Next.js dev server
- `make web-build` — build Next.js for production
- `make setup` — install all deps, run migrations, bootstrap

## What This Does NOT Include

- Multi-tenancy / tenant_id
- Password reset / email infrastructure
- User management UI
- Multiple roles (only owner)
- Background jobs / River
- Any business logic beyond auth
- Deployment configuration (K8s, Helm)
